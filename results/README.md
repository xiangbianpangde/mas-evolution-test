## 目录结构

每个版本有独立的目录：

```
results/
├── v31/                  # 冠军版本
│   ├── scores.json       # 评测分数
│   ├── composite.json    # 综合评分详情
│   ├── core_scores.json  # Core 任务分数
│   ├── gen_scores.json   # Gen 任务分数
│   └── answers/          # 各任务生成结果 (未来)
│       ├── core_001.md
│       ├── core_002.md
│       └── ...
├── v32/
├── v33/
└── v37/
```

## 当前问题

**答案文件缺失**: 当前 harness 只保存分数，不保存生成的答案。

**解决方案**: 需要修改 harness，在评测时同时保存：
- 原始问题
- 生成的答案
- 评分详情

这样才能：
1. 复现结果
2. 分析错误
3. 改进迭代
