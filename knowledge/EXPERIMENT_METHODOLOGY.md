# Experiment Methodology - 实验方法论

## 核心原则

### 1. 实验设计基础

**AB 测试原则**：
- 每次只改变一个变量
- 必须有对照组（baseline）
- 结果必须可复现

**变量控制**：
| 变量 | v31_0 baseline | 测试组 |
|------|-----------------|--------|
| tokens | 5000 | ? |
| max_runs | 2 | ? |
| temperature | 0.7 | ? |
| reflect | core_only | ? |

### 2. 评分系统

**三维评分**：
- **Depth (1-5)**: 技术深度
- **Completeness (1-5)**: 完整性
- **Actionability (1-5)**: 可操作性

**质量分计算**：
```
raw_score = Depth × Completeness × Actionability
normalized = raw_score / 125 × 100
```

**Composite 计算**：
```
composite = core_avg × 0.45 + gen_avg × 0.45 + actionability_avg × 10 × 0.1
```

### 3. 策略参数空间

**Tokens**:
- 临界值：5000（低于此质量下降）
- 6000：边际递减
- 3000：严重质量损失

**Max Runs**:
- 1：低成本但低质量
- 2：平衡（v31_0 使用）
- 3：更好的结果选择

**Temperature**:
- 0.3：确定性高
- 0.5：平衡
- 0.7：创造性
- 0.9：高创造性但可能不稳定

**Self-Reflect**:
- none：对代码任务最好
- core_only：对核心研究任务有效
- all：一般不需要

### 4. 实验分类

**探索性实验**：
- 目标：发现新策略
- 方法：小规模试点
- 样本：1-3次运行

**验证性实验**：
- 目标：确认假设
- 方法：统计显著性测试
- 样本：5+次运行

**基准实验**：
- 目标：建立标准
- 方法：完整 15 任务
- 样本：至少 3 次独立运行

### 5. 结果分析

**显著性判断**：
| 变化 | 判断 |
|------|------|
| < ±2% | 无显著差异 |
| ±2-5% | 小幅变化 |
| > ±5% | 显著变化 |
| > ±10% | 重大变化 |

**失败分类**：
- 系统错误：代码问题 → 立即修复
- 策略错误：假设不成立 → 调整方向
- 意外成功：未知因素 → 深入分析

### 6. 文档要求

**每个实验必须记录**：
```
{
  "假设": "...",
  "变量": {...},
  "结果": {...},
  "分析": "...",
  "结论": "...",
  "下次实验": "..."
}
```
