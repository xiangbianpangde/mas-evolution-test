# Human-Centric Design Principles

## 核心原则

**AI vs Human 查找方式**

| 角色 | 查找方式 | 优化方向 |
|------|---------|---------|
| **AI** | 语义/关键词搜索 | 文件名精确，内容可搜索 |
| **Human** | 视觉层级 + 关联感知 | 目录结构合理，分组清晰 |

## 设计原则

### 1. 渐进式披露 (Progressive Disclosure)
- 先展示摘要/概览
- 需要时再展开细节
- 不要一次性展示所有信息

### 2. 视觉分组 (Visual Grouping)
- 相关信息放一起
- 用空格/标题/边框区分区块
- 符合人类的空间认知

### 3. 一致性模式 (Consistent Patterns)
- 相同场景用相同布局
- 降低学习成本
- 位置/动作可预期

### 4. 反馈及时 (Prompt Feedback)
- 操作后立即显示结果
- 进度/状态可视化
- 错误信息清晰可行动

### 5. 错误可恢复 (Error Recovery)
- 重要操作可撤销
- 删除的东西在 trash
- 误操作不会造成不可逆后果

### 6. 认知负载管理 (Cognitive Load)
- 单屏信息适量
- 复杂任务分步骤
- 不要让人记忆太多上下文

## 本项目应用

### 文件架构
- 扁平结构 → 语义分组
- 用目录代表概念类别
- 子目录 README 辅助导航

### 输出格式 (待改进)
- 给人类的报告 → Markdown + 视觉层次
- 给AI的日志 → 结构化 JSON

### 演进汇报 (待改进)
- 数字表格 → 可视化图表
- 进度条 → 图形化状态
