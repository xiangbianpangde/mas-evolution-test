# Memory System - 长期记忆系统

## 1. 三层记忆架构

```
┌─────────────────────────────────────────────────────────────┐
│                     WORKING MEMORY                            │
│  (当前Session - 保存在上下文)                                 │
│  - 当前任务状态                                             │
│  - 近期决策                                                 │
│  - 活跃Agent状态                                           │
└─────────────────────────────────────────────────────────────┘
                              ↓
                              ↓ 写入文件
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    SHORT-TERM MEMORY                          │
│  (Daily Logs - memory/YYYY-MM-DD.md)                       │
│  - 当天所有实验记录                                          │
│  - 事件日志                                                 │
│  - 临时发现                                                 │
└─────────────────────────────────────────────────────────────┘
                              ↓
                              ↓ 每日归档
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                     LONG-TERM MEMORY                          │
│  (Knowledge Base - knowledge/*.md)                           │
│  - 经验教训 (LESSONS.md)                                     │
│  - 有效策略 (STRATEGIES.md)                                 │
│  - 失败模式 (ANTI_PATTERNS.md)                              │
│  - 系统状态 (STATE.md)                                      │
└─────────────────────────────────────────────────────────────┘
```

## 2. 记忆写入时机

### Working Memory (自动)

- 每个决策时
- 每个任务开始/完成时
- 每次错误发生时

### Short-term Memory (定时)

- 每小时自动保存状态
- 每天结束时归档
- 每次重要发现时

### Long-term Memory (事件触发)

- 实验完成时（记录结果）
- 发现新模式时（更新知识库）
- 犯错误时（记录教训）
- 系统变更时（更新文档）

## 3. 记忆检索机制

### 文件映射

| 信息类型 | 存储位置 | 检索方式 |
|----------|----------|----------|
| 日常日志 | memory/YYYY-MM-DD.md | 按日期 |
| 实验结果 | results/benchmarks/ | 按版本 |
| 经验教训 | knowledge/LESSONS.md | 关键词搜索 |
| 有效策略 | knowledge/STRATEGIES.md | 关键词搜索 |
| 系统架构 | knowledge/ARCHITECTURE.md | 导航 |
| 团队设计 | knowledge/TEAM_DESIGN.md | 导航 |

### 检索流程

```
查询 → 解析意图 → 定位文件 → 读取 → 返回
```

## 4. 记忆更新策略

### 冲突解决

| 冲突类型 | 解决策略 |
|----------|----------|
| 同一事实矛盾 | 以最新时间戳为准 |
| 不同经验矛盾 | 保留两条，标记分歧 |
| 旧 vs 新策略 | 新策略需明确说明旧策略为何失效 |

### 遗忘策略

- **30天** 未访问的 daily log → 压缩归档
- **90天** 未引用的经验 → 标记为历史
- **重要结论** → 永久保留

## 7. 启动流程

每次 Session 开始时：

1. 读取 `memory/today.md`（如存在）
2. 读取 `memory/最近两天.md`
3. 检查 `knowledge/STATE.md` 系统状态
4. 检查是否有未完成的任务
5. 根据状态决定下一步

## 8. 关闭流程

每次 Session 结束时：

1. 更新 `memory/today.md`
2. 如超过 24 小时，归档到 `memory/YYYY-MM-DD.md`
3. 更新 `knowledge/STATE.md`
4. 提交所有变更到 GitHub
